# Changelog

## [0.3.0] — 2026-05-30

### Changed — 推算框架重构
- **分析链路从"七层猜测"改为"四层递进约束求解"**：
  - L0 起卦 / L1 卦画默观 / L2 卦脉解构 → 物性约束向量
  - **L3 约束求解（新增核心层）**：类空间生成 + 置信度分级 + 辨识问答 + 加权判定
  - L4 变复校验
- **交互从五阶段改为三阶段 + 可旁路**：
  - S1 第一印象（卦脉直读 + 置信度分级）
  - S2 辨识问答（卦象缺口维度提问，最多2轮）
  - S3 揭晓与复盘
- **新增置信度分级输出**：🟢高置信（直接猜）/ 🟡中置信（进入问答）/ 🔴低置信（建议给提示）
- **新增提示机制**：用户可给模糊提示辅助聚焦

### Added
- `references/辨识问题库.md`：按十大品类预设辨识问题模板（二分法、排除逻辑）
- `E:/qclaw/workspace-diguan/射覆映射手册.md` v0.3：
  - 新增 §九 辨识问题设计原则（六大缺口维度、品类差异化）
  - 新增 §十 交互提问策略（流程图、置信度判定、排除逻辑、提示响应）

### Rationale
- 五轮射覆实战验证：功能态/形态推测较准，但具体物品仅1/5完全命中
- 根因：卦象信息量有限（~8维约束 vs 物品~20维特征）→ 需要辨识问题补充缺口维度
- 方向：从"AI独自猜"变为"AI给约束 + 用户帮聚焦"的联合推断模式

## [0.2.1] — 2026-05-28

### Changed
- **起卦引导流程重写**：不再让用户选择起卦方式，不再让用户手摇铜钱（除非明确指定）
- **默认起卦方式改为大衍筮法**：新增 `scripts/qigua_dayan.py`，模拟蓍草四营十八变
- 起卦前引导改为委婉确认需求（"心里想的事已经清楚了吗？给几个关键词就好"）
- 用户任何回应都视为触发起卦信号，Agent 静默执行计算
- 六爻纳甲摇卦仅当用户明确指定"铜钱""硬币""摇卦"时使用
- 梅花易数仅当用户明确指定"梅花""梅易""数字""时间""物象"时使用

### Added
- 大衍筮法起卦引擎（`scripts/qigua_dayan.py`）：49根蓍草随机分堆，四营三变成一爻，十八变成一卦
