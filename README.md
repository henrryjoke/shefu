# 卦脉解读 · 正易心法

> 「谛观」出品 · 基于麻衣道者《正易心法》42章·陈抟注

## 一句话

以正易心法卦脉法从六爻卦画中读取气的态势，**核心管道领域无关，落处可插拔扩展**——同一个卦，可以猜物，也可以问事、看考试、看天气、看行情。

## 架构

```
                ┌─→ 射覆落处 → 物品特征画像（最成熟）
起卦 → 卦象分析 ──┼─→ 问事落处 → 态势推演画像
                ├─→ 考试落处 → 考试态势画像
                ├─→ 天气落处 → 气象特征画像
                └─→ 市场落处 → 市场态势画像
```

| 层 | 领域相关性 | 说明 |
|---|---|---|
| L0 起卦 | ❌ 无关 | 大衍/梅花/六爻 |
| L1 卦画默观 | ❌ 无关 | 阴阳排列→气的态势 |
| L2 卦脉解构 | ❌ 无关 | 功能态+结构信号 |
| **L3 落处映射** | ✅ **领域相关** | 气的态势→特定域类象→特征/态势画像 |
| L4 变复校验 | ❌ 无关 | 爻变+体复+参卦 |

## 触发词

| 你说 | 落处 | AI做 |
|------|------|------|
| 「射覆」「猜一猜」 | 射覆 | 起卦→物品特征画像 |
| 「问事」「推演一下」 | 问事 | 起卦→态势推演画像 |
| 「问考试」 | 考试 | 起卦→考试态势画像 |
| 「看天气」 | 天气 | 起卦→气象特征画像 |
| 「看股票」 | 市场 | 起卦→市场态势画像 |

## 核心理念

**AI 是画师不是鉴定师**——刻画特征画像/态势画像，判断权始终在你手中。

- **不判断吉凶**——只展开可能性
- **不替你命名/决定**——画像给你参考，你自己判断
- **同构容纳**——同一气脉在自然、人身、器物、人伦有同构落处

## 射覆（最成熟落处）

六维度渐进精化：材质 / 形态 / 功能 / 结构 / 颜色 / 尺寸

```
S1 第一印象 → S2 渐进精化（可多轮） → S3 揭晓与复盘
```

- 类象库：`data/类象库.json`（结构化信号→特征映射）
- 用户画像：`profile.json`（特征维度偏好+场景先验）
- 三层自我进化：即时学习 + 用户画像 + 类象库自动更新

## 问事（新增落处）

四维态势推演：走向 / 矛盾 / 转折 / 条件

```
S1 态势初判 → S2 多维展开 → S3 用户反馈精化
```

## 目录结构

```
shefu/
├── SKILL.md           # 主技能定义（多落处架构）
├── config.json        # 触发词+元数据
├── _meta.json         # 发布元数据
├── scripts/           # 起卦引擎（领域无关）
│   ├── qigua.py       # 起卦入口
│   ├── qigua_dayan.py # 大衍筮法
│   ├── qigua_meihua.py# 梅花易数
│   ├── qigua_liuyao.py# 六爻纳甲
│   ├── lifa.py        # 历法工具
│   └── update_kb.py   # 类象库更新
├── data/
│   ├── 类象库.json    # 射覆类象知识库
│   ├── guaci_db.json  # 64卦基础数据
│   ├── najia_table.json# 纳甲数据
│   └── 落处/          # 落处定义（可插拔）
│       ├── 射覆.json
│       ├── 问事.json
│       ├── 考试.json
│       ├── 天气.json
│       └── 市场.json
├── references/        # 参考文献目录
├── coze/              # Coze 平台适配
│   ├── prompt_system.md
│   └── setup_guide.md
└── profile.json       # 用户画像
```
