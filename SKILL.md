## 🧙 什么是卦脉解读？

本技能继承五代宋初隐士**麻衣道者**《正易心法》及**陈抟**（希夷先生）注的卦脉解读传统，以「爻位即气位」为核心方法，**不查卦辞、不套口诀**，从六爻卦画的阴阳排列中直接读取气的态势。

> **核心理念**：同一气脉可落于多个维度——**一脉多现**。气的态势是领域无关的，落处是可插拔的。

```
                ┌─→ 射覆落处 → 物品特征画像
起卦 → 卦象分析 ──┼─→ 问事落处 → 态势推演画像
                ├─→ 考试落处 → 考试态势画像
                ├─→ 天气落处 → 气象特征画像
                └─→ 市场落处 → 市场态势画像
```

### 落处信号表

| 你说 | 落处 | AI回应 | 启动信号 | AI执行 |
|------|------|--------|---------|--------|
| 「玩射覆」「射覆」「猜一猜」「帮我猜」 | 射覆 | 「好，我已准备好起卦。」 | 「开始」 | 大衍筮法起卦→物品特征画像 |
| 「问事」「解卦」「推演一下」「算一卦」 | 问事 | 「心里想的事清楚了吗？给几个关键词就好。」 | 任何回应 | 大衍筮法起卦→态势推演画像 |
| 「问考试」「考试运势」 | 考试 | 「告诉我考试类型和日期。」 | 任何回应 | 大衍筮法起卦→考试态势画像 |
| 「看天气」「明天天气」 | 天气 | 「告诉我要看哪天的天气。」 | 任何回应 | 大衍筮法起卦→气象特征画像 |
| 「看股票」「问行情」 | 市场 | 「告诉我要看哪个标的/板块。」 | 任何回应 | 大衍筮法起卦→市场态势画像 |

**核心规则**：

- **调用信号** = 告诉 AI「我要做什么」+「落到哪个域」
- **启动信号** = 告诉 AI「可以开始了」（AI 才会执行起卦）
- **没有启动信号，AI 不起卦**——不会在你说触发词后就自作主张
- 你不说起卦方式 → 默认大衍筮法

## 🧠 核心方法（三分钟入门）

### 原则一：爻位即气位

阳爻在哪里，阳气就在哪里；阴爻在哪里，阴气就在哪里。**不看标签，看位置。**

### 原则二：八卦是八种功能态

八卦不是名词，是德性（功能态）。从卦画结构直接读：

| ☰ 乾 | ☷ 坤 | ☳ 震 | ☴ 巽 | ☵ 坎 | ☲ 离 | ☶ 艮 | ☱ 兑 |
|---|---|---|---|---|---|---|---|
| 三阳 | 三阴 | 一阳**下** | 一阴**下** | 一阳**中** | 一阴**中** | 一阳**上** | 一阴**上** |
| **健** | **顺** | **动** | **入** | **陷** | **丽** | **止** | **悦** |

### 原则三：观不是推

先静观卦画结构——六爻阴阳怎么排列，气就怎么走。**观是直观照见，不是逻辑推导。**

### 原则四：同构容纳

同一气脉可落于多个维度——自然、人身、器物、人伦。**一脉多现。**

### 原则五：合于自然者为正

每个推演必须能从卦画结构回溯验证。方向不可逆：**结构 → 自然 → 语境**，不能从语境倒推。

## 📐 分析链路（核心引擎 + 落处映射）

### 核心引擎（领域无关）

```
L0 起卦 → 大衍筮法（默认）/ 梅花易数 / 六爻纳甲

L1 卦画默观
  不看卦名，只看六爻阴阳排列。
  · 上卦下卦各是什么功能态（健/顺/动/入/陷/丽/止/说）
  · 阳爻在哪里，阴爻在哪里——气就怎么走
  · 重卦判断：上下卦相同 → 双体/对称结构信号

L2 卦脉解构
  从功能态 + 结构信号 → 提取气的态势特征。
  · 反对分析：本卦→变卦揭示核心动态
  · 卦脉流动：阴阳进退、消息定位、互体表里
  · 月建加权：旺相休囚死 → 强弱/时令调节
  · 动爻定位：动爻位置 → 变化部位
  · 颜色系统：月建→外壳/外围色；日建→核心/内部色

  → 输出气的态势画像（领域无关的中间表示）：
    功能态组合 + 动态特征 + 结构信号 + 动爻位置 + 月建日建
```

### 落处映射（领域相关）

L3 是**落处映射层**——将领域无关的气的态势投射到特定域。

#### 落处1：射覆（猜物）

```
L3 射覆映射 ⭐（当前最成熟）

3.0 读取画像与类象库（每轮执行前）
    读取 profile.json → 获取特征维度偏好 + 场景先验
    读取 data/类象库.json → 获取信号→特征映射置信度

3.1 画像展开与交叉验证
    对画像 v1 每个维度，用类象库交叉验证：
    · 单卦信号 → 直接映射（如艮→硬质）
    · 组合信号 → 交叉推断（如艮+坎互体→硬质+含锋利部件）
    · 重卦信号 → 双体对称通则（如艮重卦→双体对称结构）
    · 动爻信号 → 变化部位定位（如上爻动→顶端可开合）
    用户偏好加权：历史命中率高的维度 ×1.2。

3.2 置信度分级
    🟢 高置信：卦象信号直接支持，类象库验证 ≥0.75
    🟡 中置信：信号支持但映射有多种可能，类象库 0.40-0.74
    🔴 低置信：信号模糊或类象库无记录 <0.40

3.3 补充性提问（仅S2触发）
    对 🟡 中置信维度，问1-2个卦象无法给出的补充问题：
      大小：掌心级 / 桌面级 / 地面级
      重量：轻飘飘 / 有分量 / 沉手
      触感：光滑 / 粗糙 / 软 / 硬
      场景：桌上 / 口袋 / 手里 / 包里
    规则：问题二选一，最多2个问题后必须更新画像

3.4 画像更新
    根据反馈/回答 → 更新特征画像 → v2, v3...
    每轮更新标注变化处

射覆维度：材质 / 形态 / 功能 / 结构 / 颜色 / 尺寸
射覆类象库：data/类象库.json
```

## 🎮 射覆玩法说明

### 经典玩法（二人射覆）

```
玩家A：「起一卦，猜我手里握着什么。」
AI：起卦 → 默观卦画 → 给出特征画像 → 根据反馈渐进精化
玩家A：对照画像与真实物品，自行判断
```

### 单人玩法（自测卦脉）

```
玩家：「起卦猜我左手边的杯子里有什么。」
AI：起卦 → 给出物性画像 → 玩家可说「继续」让AI深入某个维度
玩家：对照实际物品，看画像哪些维度命中
```

### 提示机制（辅助聚焦）

```
「提示：它和电力有关」  → AI 沿「坎=电」方向深化细节
「提示：跟我头有关」    → AI 搜索「乾=首，艮=鼻，兑=耳，离=目」
「提示：它比较小」      → AI 调整尺寸维度，排除大件
```

### 射覆交互流程：渐进精化

```
S1 — 第一印象（卦脉直读）
  AI 给出：
    · 卦象总览（本卦/变卦/动爻）
    · 功能态直读（卦的结构在说什么）
    · 特征画像 v1（各维度描述 + 置信度 🟢🟡🔴）
    · 同信号参考（历史上类似信号组合出现过什么——供你联想，非候选）
  
  你可以：
    → 说「对了」（方向正确，进入S2精化细节）
    → 说「偏了」（方向有误，给个模糊提示帮助调整）
    → 说「接近」（方向对但某个维度不对，告诉AI哪个）
    → 直接揭晓答案（任何时候都可以）

S2 — 渐进精化（可多轮）⭐ 核心交互
  AI 根据你的反馈，做以下之一：
  · 方向正确 → 深入展开匹配度高的维度
  · 方向偏了 → 根据提示调整画像
  · 补充卦象看不到的维度（问1-2个补充性问题）
  → 每轮更新特征画像，越来越精确

S3 — 揭晓与复盘
  ① 特征回溯
  ② 画像更新（profile.json）⭐ 每轮必执行
  ③ 类象库更新（data/类象库.json）⭐ 每轮必执行
  ④ 输出复盘摘要
  ⑤ 询问是否补充背景习惯（可选）
```

## ⚠️ 使用须知

- **不判断吉凶**——不给出确定性结论，只展开可能性
- **不替你决定**——AI 刻画态势/画像，你结合实际判断
- **禁用语**：一定、必然、大吉、大凶、宜、忌、应该
- **鼓励用语**：可能、或许、含有…之象、呈现…态势
- **结束语**——"以上是根据卦象展开的态势画像，并非确定性结论。请你结合实际情况判断。"
