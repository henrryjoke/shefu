# 射覆 · 正易心法卦脉解读

> 「谛观」出品 · 基于麻衣道者《正易心法》42章·陈抟注

---

## 🧙 什么是射覆？

**射覆**（Shefu）——「射」者猜也，「覆」者盖也。一人藏物于覆器之下，另一人起卦推测覆下为何物。庄子曰：

> 射者非前期而中，谓之善射。  
> ——《庄子·田子方》

射覆见载于《汉书·东方朔传》。汉武帝尝以射覆试东方朔，朔起卦皆中，帝叹服。自此射覆成为汉代宫廷方士、文人之间的流行占卜游戏——**既是易学的功夫试炼，也是智力的雅趣对决。**

本技能继承五代宋初隐士**麻衣道者**《正易心法》及**陈抟**（希夷先生）注的卦脉解读传统，以「爻位即气位」为核心方法，**不查卦辞、不套口诀**，从六爻卦画的阴阳排列中直接读取气的态势，**渐进精化物品的特征画像**——AI 画出画像，你对照真实场景判断。

> **核心理念**：AI 是你的「画师」，不是「鉴定师」。AI 的任务是刻画出越来越精确的特征画像，命名权始终在你手中。

## 🎮 玩法说明

### 经典玩法（二人射覆）

```
玩家A：「起一卦，猜我手里握着什么。」
AI：起卦 → 默观卦画 → 给出特征画像 → 根据反馈渐进精化
玩家A：对照画像与真实物品，自行判断
```

### 单人玩法（自测卦脉）

```
玩家：「起卦猜我左手边的杯子里有什么。」
AI：起卦 → 给出物性画像 → 玩家可说「继续」让AI深入某个维度
玩家：对照实际物品，看画像哪些维度命中
```

### 无限定玩法（开放式）

```
玩家：「起一卦，随便猜一个我家里的东西。」
AI：起卦后从卦脉结构提取特征画像，
    给出各维度的描述与置信度，你自行对照。
```

### 提示机制（辅助聚焦）

如果你觉得画像方向对但不够精确，可以给一个**模糊提示**（不说具体物品）：

```
「提示：它和电力有关」  → AI 沿「坎=电」方向深化细节
「提示：跟我头有关」    → AI 搜索「乾=首，艮=鼻，兑=耳，离=目」
「提示：它比较小」      → AI 调整尺寸维度，排除大件
```

## 🧠 核心方法（三分钟入门）

### 原则一：爻位即气位

阳爻在哪里，阳气就在哪里；阴爻在哪里，阴气就在哪里。**不看标签，看位置。**

### 原则二：八卦是八种功能态

八卦不是名词，是德性（功能态）。从卦画结构直接读：

| ☰ 乾 | ☷ 坤 | ☳ 震 | ☴ 巽 | ☵ 坎 | ☲ 离 | ☶ 艮 | ☱ 兑 |
|---|---|---|---|---|---|---|---|
| 三阳 | 三阴 | 一阳**下** | 一阴**下** | 一阳**中** | 一阴**中** | 一阳**上** | 一阴**上** |
| **健** | **顺** | **动** | **入** | **陷** | **丽** | **止** | **悦** |

### 原则三：观不是推

先静观卦画结构——六爻阴阳怎么排列，气就怎么走。**观是直观照见，不是逻辑推导。**

### 原则四：同构容纳

同一气脉可落于多个维度——自然、人身、器物、人伦。**一脉多现。**

### 原则五：合于自然者为正

每个推演必须能从卦画结构回溯验证。方向不可逆：**结构 → 自然 → 语境**，不能从语境倒推。

## 🔮 调用与起卦

### 两种场景，两套信号

| 场景 | 调用信号（你说） | AI回应 | 你的启动信号 | AI执行 |
|------|----------------|--------|-------------|--------|
| **射覆（猜物）** | 「玩射覆」「射覆」「猜一猜」「帮我猜」 | 「好，我已准备好起卦。」 | 「开始」 | 大衍筮法起卦 |
| **问事（解卦）** | 「起卦」「测算」「算一算」「帮我算」「问卦」 | 「心里想的事清楚了吗？给几个关键词就好。」 | 任何回应（关键词/一句话/甚至「嗯」） | 大衍筮法起卦 |

**核心规则**：

- **调用信号** = 告诉 AI「我要做什么」（射覆还是问事）
- **启动信号** = 告诉 AI「可以开始了」（AI 才会执行起卦）
- **没有启动信号，AI 不起卦**——不会在你说"起卦"后就自作主张
- 你不说起卦方式 → 默认大衍筮法（不让你选、不让你摇）

### 射覆交互流程：渐进精化

```
S1 — 第一印象（卦脉直读）
  AI 给出：
    · 卦象总览（本卦/变卦/动爻）
    · 功能态直读（卦的结构在说什么）
    · 特征画像 v1（各维度描述 + 置信度 🟢🟡🔴）
    · 同信号参考（历史上类似信号组合出现过什么——供你联想，非候选）
  
  你可以：
    → 说「对了」（方向正确，进入S2精化细节）
    → 说「偏了」（方向有误，给个模糊提示帮助调整）
    → 说「接近」（方向对但某个维度不对，告诉AI哪个）
    → 直接揭晓答案（任何时候都可以）

S2 — 渐进精化（可多轮）⭐ 核心交互
  AI 根据你的反馈，做以下之一：
  
  · 方向正确 → 深入展开匹配度高的维度
    （互体/变卦/月建等未用尽的信号源 → 更细的特征描述）
  
  · 方向偏了 → 根据提示调整画像
    （重新审视卦象信号，调整维度权重）
  
  · 补充卦象看不到的维度（问1-2个补充性问题）：
    大小：掌心级 / 桌面级 / 地面级
    重量：轻飘飘 / 有分量 / 沉手
    触感：光滑 / 粗糙 / 软 / 硬
    场景：桌上 / 口袋 / 手里 / 包里
  
  → 每轮更新特征画像，越来越精确
  
  你可以：
    → 说「继续」（再深入一层）
    → 说「够了」（画像足够精确，你已判断出来）
    → 给更具体的提示
    → 随时揭晓

S3 — 揭晓与复盘
  ① 特征回溯
    · 画像哪些维度命中了？（记录到 profile.json）
    · 哪些偏了？（分析原因：信号误读 / 权重偏移 / 维度缺失）
    · 哪些维度对玩家判断帮助最大？
  
  ② 画像更新（profile.json）⭐ 每轮必执行
    · 特征维度偏好：命中维度权重 +0.08（上限 2.0）
                     遗漏但存在的维度权重 +0.03
                     错误维度权重 -0.10（下限 0.3）
    · 场景先验：本轮场景类型 count+1
    · 最近 20 轮：追加本轮记录
    · 全局统计：更新特征命中率
  
  ③ 类象库更新（data/类象库.json）⭐ 每轮必执行
    · 命中信号→特征映射：confidence += 0.05（上限 0.95）
    · 新发现的特征关联：新增条目，confidence = 0.30
    · 遗漏但存在的信号：记录为关联，confidence += 0.02
    · 注意：首次运行需手动评估本轮结果后更新
  
  ④ 输出复盘摘要
    · 画像 v1→v最终 的变化轨迹
    · 哪些维度从 🟡→🟢（玩家反馈帮助了）
    · 哪些维度始终 🟡 或 🔴（卦象信息不足的维度）
    · 类象库有什么更新？
  
  ⑤ 询问是否补充背景习惯（可选）
```

## 📐 分析链路（四层特征画像）

```
L0 起卦 → 大衍筮法（默认）/ 梅花易数 / 六爻纳甲

L1 卦画默观
  不看卦名，只看六爻阴阳排列。
  · 上卦下卦各是什么功能态（健/顺/动/入/陷/丽/止/说）
  · 阳爻在哪里，阴爻在哪里——气就怎么走
  · 重卦判断：上下卦相同 → 双体/对称结构信号

L2 卦脉解构
  从功能态 + 结构信号 → 提取特征画像 v1。
  · 反对分析：本卦→变卦揭示物品的核心动态特征
  · 卦脉流动：阴阳进退、消息定位、互体表里
  · 月建加权：旺相休囚死 → 材质/颜色时令调节
  · 动爻定位：动爻位置 → 变化部位（见注1）
  · 颜色系统：月建→外壳/手柄色；日建→功能部/内部色
  
  → 输出特征画像 v1：
    材质 🟢🟡🔴  描述 + 卦象依据
    形态 🟢🟡🔴  描述 + 卦象依据
    功能 🟢🟡🔴  描述 + 卦象依据
    结构 🟢🟡🔴  描述 + 卦象依据
    颜色 🟢🟡🔴  描述 + 卦象依据
    尺寸 🟢🟡🔴  描述 + 卦象依据
  
  → 同信号参考：类似信号组合在历史中出现的物品（供联想，非候选）

L3 特征精化 ⭐（核心层）
  3.0 读取画像与类象库（每轮执行前）
      读取 profile.json → 获取特征维度偏好 + 场景先验
      读取 data/类象库.json → 获取信号→特征映射置信度
      （这些权重影响 3.1 画像各维度的优先级）

  3.1 画像展开与交叉验证
      对画像 v1 每个维度，用类象库交叉验证：
      · 单卦信号 → 直接映射（如艮→硬质）
      · 组合信号 → 交叉推断（如艮+坎互体→硬质+含锋利部件）
      · 重卦信号 → 双体对称通则（如艮重卦→双体对称结构）
      · 动爻信号 → 变化部位定位（如上爻动→顶端可开合）
      用户偏好加权：历史命中率高的维度 ×1.2。

  3.2 置信度分级
      🟢 高置信：卦象信号直接支持，类象库验证 ≥0.75
         → 直接输出描述，不需要补充信息
      🟡 中置信：信号支持但映射有多种可能，类象库 0.40-0.74
         → 给出描述 + 可能在S2补充此维度
      🔴 低置信：信号模糊或类象库无记录 <0.40
         → 给出推测方向 + 建议玩家给提示
  
  3.3 补充性提问（仅S2触发）
      对 🟡 中置信维度，问1-2个卦象无法给出的补充问题：
        大小：掌心级 / 桌面级 / 地面级
        重量：轻飘飘 / 有分量 / 沉手
        触感：光滑 / 粗糙 / 软 / 硬
        场景：桌上 / 口袋 / 手里 / 包里
      
      规则：
        · 问题二选一（帮AI聚焦，不是让你盲猜）
        · 最多2个问题后必须更新画像
        · 你任何时候可以说「够了」跳过问答
  
  3.4 画像更新
      根据反馈/回答 → 更新特征画像 → v2, v3...
      每轮更新标注变化处（哪些维度从🟡→🟢）

L4 变复校验
  爻变 + 体复 + 参卦（自校验）。
  · 验证特征推导一致性，排除矛盾
  · 生成复盘用的回溯路径

注1：动爻位置与物理对应
  初爻=底部/根基  二爻=下部  三爻=中下部
  四爻=中上部      五爻=上部  上爻=顶部/尖端/最外层
  阳→阴动=硬→软/实→虚/闭→开
  阴→阳动=软→硬/虚→实/开→闭
  ⚠️ 特别注意：动爻可能指示机械运动（铰链/折叠/开合/旋转/滑动）
```

## 🎯 输出示例（特征画像模式）

```
📋 卦象总览
  本卦：☶☶ 艮为山 · 变卦：☷☶ 地山谦
  动爻：上六阳→阴动

🔬 特征画像 v1

  材质 🟢 硬质（艮=土石/止），可能含金属部件
       依据：艮为石为土，互体坎=险/锋利→可能含锋利组件
  
  形态 🟢 双体对称，框状结构
       依据：重卦艮→双体对称，艮为门阙→框状
  
  功能 🟡 常态静止，顶端可开合
       依据：艮为止（静止），上六阳→阴动=顶端闭→开
  
  结构 🟡 有铰链或运动部件
       依据：互体震=动+坎=陷→内部有启动力+中空结构
  
  颜色 🟡 手柄偏暖色，功能部银白
       依据：月建巳火→外壳暖色，日建申金→功能部偏金属色
  
  尺寸 🟡 手持级
       依据：艮为小石→手掌可持之物

  💡 同信号参考：这类结构（硬质+双体+框状+开合）在古代见于刀剪，
     现代类似特征的有手持开合工具类。

  你觉得这个画像接近你藏的东西吗？
  → 说「对了」我继续精化细节
  → 说「偏了」给我个提示调整方向
  → 说「接近但______」我调整某个维度
```

**S2 精化后可能变成：**

```
🔬 特征画像 v2（变化处用 ← 标注）

  材质 🟢 硬质，含金属 ← 不变
  形态 🟢 双体对称，框状 ← 不变
  功能 🟢 常态静止，顶端可开合 ← 🟡→🟢（玩家确认方向正确）
  结构 🟢 铰链结构，两个对称操作件 ← 🟡→🟢（深入互体后确认）
  颜色 🟡 手柄暖色，功能部银白 ← 不变
  尺寸 🟢 手持工具级 ← 🟡→🟢（玩家提示"比较小"）

  画像已稳定在：硬质+双体框状+开合铰链+手持级
  你心里有答案了吗？
```

## ⚠️ 使用须知

- **不判断吉凶**——不给出确定性结论，只展开可能性
- **不替你命名**——AI 画特征画像，你对照真实场景判断是什么
- **禁用语**：一定、必然、大吉、大凶、宜、忌、应该
- **鼓励用语**：可能、或许、含有…之象、呈现…态势
- **结束语**——"以上是根据卦象展开的特征画像，并非确定性结论。请你结合实际场景判断。"
