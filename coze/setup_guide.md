# 卦脉解读 Coze Bot — 搭建指南

> 本文档描述将卦脉解读 Skill 从 WorkBuddy 迁移到 Coze（扣子）平台的具体步骤。

---

## 1. Bot 基本设置

| 设置项 | 值 |
|--------|-----|
| 名称 | 卦脉解读 |
| 描述 | 基于《正易心法》的卦脉解读助手。从卦画结构中直接读取气的态势，支持多落处投射（射覆猜物/问事推演/考试/天气/市场）。不查表、不套口诀。 |
| 图标 | 🧙 或上传自定义图标 |
| 模型 | 推荐 DeepSeek / 豆包（需支持长文本+结构化输出） |
| 系统提示词 | `coze/prompt_system.md` 全文 |

## 3. 工作流设计

### 3.1 起卦工作流（`shefu-qigua`）

```
Start 节点
  │ 输入: method (dayan|meihua|liuyao), params (可选)
  ▼
Code 节点: 起卦计算
  │ 粘贴 qigua.py 中的 qigua() 函数 + 所有依赖（lifa常量和工具函数）
  │ 输入: method, params
  │ 输出: result (JSON)
  │ 超时: 60秒（实际<0.1秒）
  ▼
End 节点
  │ 输出: 卦象JSON
```

**Code 节点内容**：复制 `qigua.py` 中以下内容拼成一个 Code 节点：
1. TGAN/DZHI/GANZHI_60/DZHI_WX/MONTH_BRANCH_SOLAR（干支历法常量）
2. BAGUA_PIC/BAGUA_WUXING/BAGUA_YAO/XIANTIAN 等（八卦常量）
3. WUXING_SHENGKE/LIUSHEN_ORDER/LIUSHEN_START（五行六神）
4. 工具函数（yao_pic/yao_to_gua/hexagram_view）
5. 完整起卦函数（dayan_qigua/meihua_qigua/liuyao_qigua/qigua）
6. 内联 JSON 数据（_GUACI_JSON/_NAJIA_JSON）
7. main() 函数入口

### 3.2 简化方案（推荐 V1）

如果不想做工作流，可以直接用单一 LLM Bot（无工作流）：

1. **Bot 配置**：粘贴系统提示词 + 挂载知识库
2. **用户交互**：用户说「玩射覆」「猜一猜」→ AI 起卦 → 给出特征画像
3. **限制**：无法自动计算月建旺衰，LLM 推理的数学精度有限

**V1 建议**：先用简化方案验证提示词质量 → 收集反馈 → V2 加入起卦工作流。

## 4. 测试用例

### 用例 1：射覆猜物
```
用户: 玩射覆，猜猜我手里是什么
预期: 回复"好，我已准备好起卦。"→ 用户说"开始" → 起卦 → 特征画像v1
      包含：材质🟢 硬质 / 形态🟡 双体 / 功能🟡 可开合 等维度
```

### 用例 2：问事推演
```
用户: 推演一下，我最近工作上的事情
预期: 回复"心里想的事清楚了吗？给几个关键词就好。"→ 用户说关键词 → 起卦 → 态势画像v1
      包含：走向🟢 / 矛盾🟡 / 转折🟡 / 条件🟢 等维度
```

### 用例 3：用户给模糊提示（射覆）
```
用户: (在画像v1后) 接近，它比较小
预期: 精化画像v2：尺寸维度从🟡→🟢，排除大件物品
```

### 用例 4：用户提供卦名
```
用户: 起了一卦是艮为山，上爻动，猜我藏了什么
预期: 按"艮·止（硬质/静止）+ 重卦→双体对称 + 上爻动→顶端可开合"推理
      给出特征画像，使用联想冷却
```
