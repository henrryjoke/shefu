# 射覆 Coze Bot — 搭建指南

> 本文档描述将射覆 Skill 从 WorkBuddy 迁移到 Coze（扣子）平台的具体步骤。

---

## 1. Bot 基本设置

| 设置项 | 值 |
|--------|-----|
| 名称 | 射覆 |
| 描述 | 基于《正易心法》的卦脉解读助手。从卦画结构中直接读取气的态势，不查表、不套口诀。 |
| 图标 | ☯️ 或上传自定义图标 |
| 模型 | 推荐 DeepSeek / 豆包（需支持长文本+结构化输出） |
| 系统提示词 | `coze/prompt_system.md` 全文 |

## 3. 工作流设计

### 3.1 起卦工作流（`shefu-qigua`）

```
Start 节点
  │ 输入: method (dayan|meihua|liuyao), params (可选)
  ▼
Code 节点: 起卦计算
  │ 粘贴 qigua.py 中的 qigua() 函数 + 所有依赖（lifa常量和工具函数）
  │ 输入: method, params
  │ 输出: result (JSON)
  │ 超时: 60秒（实际<0.1秒）
  ▼
End 节点
  │ 输出: 卦象JSON
```

**Code 节点内容**：复制 `qigua.py` 中以下内容拼成一个 Code 节点：
1. TGAN/DZHI/GANZHI_60/DZHI_WX/MONTH_BRANCH_SOLAR（干支历法常量）
2. BAGUA_PIC/BAGUA_WUXING/BAGUA_YAO/XIANTIAN 等（八卦常量）
3. WUXING_SHENGKE/LIUSHEN_ORDER/LIUSHEN_START（五行六神）
4. 工具函数（yao_pic/yao_to_gua/hexagram_view）
5. 完整起卦函数（dayan_qigua/meihua_qigua/liuyao_qigua/qigua）
6. 内联 JSON 数据（_GUACI_JSON/_NAJIA_JSON）
7. main() 函数入口

### 3.2 解卦工作流（`shefu-jiegua`）

```
Start 节点
  │ 输入: gua_result (卦象JSON，来自起卦工作流的输出)
  │ 输入: user_context (用户事项关键词)
  ▼
LLM 节点: 解卦分析
  │ 模型: DeepSeek / 豆包
  │ 系统提示词: prompt_system.md
  │ 用户消息模板:
  │    """
  │    请根据以下卦象进行六层递进分析：
  │    
  │    {起卦结果}
  │    
  │    用户关注事项：{用户输入}
  │    
  │    请按输出结构逐层展开分析。
  │    """
  │ 知识库: 已勾选全部8篇方法论文档
  ▼
End 节点
  │ 输出: 解卦全文
```

### 3.3 简化方案（推荐 V1）

如果不想拆两个工作流，可以直接用单一 LLM Bot（无工作流）：

1. **Bot 配置**：粘贴系统提示词 + 挂载知识库
2. **用户交互**：用户直接输入卦名/卦画/起卦请求
3. **限制**：无法自动计算月建旺衰，LLM 推理的数学精度有限

**V1 建议**：先用简化方案验证提示词质量 → 收集反馈 → V2 加入起卦工作流。

## 4. 测试用例

### 用例 1：用户提供卦名
```
用户: 乾为天卦，问事业
预期: 六层递进分析，包含"六爻纯阳""健而又健""自反对体无主客"等关键判断
```

### 用例 2：用户要求起卦
```
用户: 帮我起一卦，看看财运
预期: 回复"请稍等，正在为您起卦…" → 展示卦象 → 递进分析
      （纯LLM模式下由LLM模拟起卦；完整模式下调用Code节点）
```

### 用例 3：射覆猜物
```
用户: 起了一卦是火风鼎，猜猜我手里是什么
预期: 按"离·大腹（中空有容）+ 巽·入（渗透发散）+ 鼎→容纳调和"推理
      给出2-3个候选，使用联想冷却
```
